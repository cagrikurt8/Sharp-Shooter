Description

Our game is too simple. There are a couple of things that we can work on.

First, in the real world, laws of physics influence the movement of objects. Remember physics in middle school? Any movement has acceleration. Objects don’t start or stop instantaneously; instead, it takes time to speed up and slow down. The following figures show the difference between the movement in our program and the movement with acceleration:

So far, when we move the Gun Sight, for instance, to the right, we increase the horizontal coordinate once (x += 10) and update the screen. To simulate an accelerated motion, we have to change this algorithm. We should create a sequence of changes to the horizontal variable, for instance, (1, 2, 4, 2, 1). For each element of this collection, we should do the following operations:

  • x += 10;
  • update screen;

Second, in real life, hitting a target requires particular skills and composure. Hands are shaking, the wind is blowing, the gun can misfire. A lot of factors. Under Newton’s second law, force produces motion with acceleration. That is, you can apply the algorithm discussed above.

Above, we have discussed only one example of how you can apply the acceleration feature. You can implement any variation that you want.

Objectives

To complete the stage, please, comply with the following:

  • The requirements of constructing graphical objects remain the same as in the previous stages;
  • For testing purposes, the initial position of the Gun Sight center should be at (100, 100);
  • The main idea of this stage is to make the accuracy of shooting as hard as possible.
